# AI 研发过程文档

## 一、Prompt 迭代记录

### V1：基础指令式（初版）

```
你是一位资深小红书文案创作者，擅长写出高互动、高收藏的爆款笔记。
你的写作特点：
1. 标题吸引眼球，善用数字、emoji和悬念
2. 正文结构清晰，善用分段和小标题
3. 语言贴近年轻用户，自然不做作
4. 结尾引导互动（点赞、收藏、评论）
5. 标签精准，覆盖热门话题
```

**问题**：输出质量不稳定，字数常超标，部分结果缺少互动引导。

---

### V2：增加硬约束 + 读者画像

```
你是一位资深小红书文案创作者，擅长写出高互动、高收藏的爆款笔记。
你的目标读者是18-35岁的年轻用户，以女性为主。
（新增）
3. 避免过于专业的术语（如需使用请加通俗解释）
4. 结尾必须有互动引导语
6. 严格遵守用户要求的字数范围，不得超出
```

User Prompt 新增：
```
【字数硬限制】正文必须在X字之间，不可超出

## 互动引导（1句话，引导点赞/收藏/评论）
```

**效果**：字数控制明显改善，互动引导不再缺失。

---

### V3：Few-shot + 爆款公式体系

**核心改进：从"告诉模型怎么做"升级为"给模型看真实案例"**

```
你是一位年收入百万的小红书头部博主，精通爆款内容创作方法论。
你深谙小红书算法推荐机制和用户心理。

## 爆款标题公式（任选其一）
- 数字+结果："3步搞定XX"
- 反差+悬念："被骂了3年，终于翻红的XX"
- 身份+痛点："打工人必看！XX神器"
- 情绪+共鸣："谁懂啊！XX真的绝了"

## 写作铁律
- 每一句都要有"获得感"或"情绪价值"
- 绝不写空洞无信息量的废话
```

新增 Few-shot 真实案例注入：
```
以下是几个高赞小红书文案的真实案例，请学习它们的写作模式：
【案例1 - 种草推荐】（点赞12300）
标题：姐妹们！这个底妆搭配让我省了2000💰
...
```

**效果**：输出风格显著接近真实爆款，标题公式化、正文信息密度高。

---

### V4（当前版本）：Agent 架构 + 动态检索

**核心改进：从"固定 Prompt"升级为"动态检索 + 多轮迭代 + 个性化学习"**

System Prompt 改为动态生成：
- 根据用户输入的主题，从案例库检索最相关的 3 篇爆款
- 将检索结果（含真实互动数据）注入 Prompt 上下文
- 读取用户历史偏好，融入系统提示

新增精修 Prompt（禁止输出元信息）：
```
【重要要求】
- 只输出优化后的完整文案
- 禁止输出任何修改说明、修改记录、优化备注
- 禁止出现"删除""新增""修改""从X→Y"等元描述文字
- 直接给出最终文案，就像这是你第一次写的一样
```

## 二、AI 协作过程

### 2.1 协作模式

本项目采用"人类提需求 + AI 实现"的协作模式：
- 人类角色：产品经理（提出需求、评审质量、反馈问题）
- AI 角色：全栈开发者（技术选型、编码实现、测试优化）

### 2.2 协作关键节点

| 阶段 | 人类输入 | AI 输出 |
|------|---------|---------|
| 产品选型 | "帮我选方向" + 约束条件 | 推荐"小红书文案生成器"并说明理由 |
| 需求对齐 | 回答 AI 的追问（技术栈/交付形式/时间/API） | 完整方案 + 项目结构设计 |
| V1 开发 | "开始搭建" | 可运行的 MVP 代码 |
| V1 测试 | 提供 API Key | 3组测试 + 问题记录 |
| V2 优化 | "字数超标/缺互动引导" | Prompt 修改 + 验证测试 |
| V3 重构 | "全方位提升" | Few-shot + 公式体系 + UI重构 |
| 自我批判 | "这真的好吗？" | 6个维度的诚实自评 |
| Agent 升级 | "都需要升级" | 案例库 + 多轮迭代 + 规则评分 + 持久化 |
| 体验打磨 | "用户不需要看这些" | 移除技术细节，纯C端界面 |
| 流程优化 | "展示思考过程" | 右侧实时显示检索/评分/优化步骤 |

### 2.3 关键决策记录

**决策1：产品方向选择**
- AI 建议"小红书文案生成器"，理由：用户场景清晰、技术架构简单、Prompt工程是核心
- 人类同意

**决策2：用模拟数据代替实时爬虫**
- AI 提问：小红书反爬严格，Demo中实时爬取不现实，可否用本地JSON模拟？
- 人类选择：模拟数据

**决策3：前端不暴露技术实现**
- 人类反馈：API Key输入太low / 评分数据用户不关心 / Agent迭代过程不该暴露
- AI 逐步移除所有技术信息，界面纯面向C端用户

**决策4：思考过程的展示粒度**
- 人类反馈：右侧白屏太久，要展示AI思考过程
- AI 最终方案：展示检索结果/质量检测/薄弱项分析，但不展示原始Prompt和代码逻辑

## 三、技术架构演进

```
V1: 用户输入 → 单次API调用 → 输出
V2: 用户输入 → 优化Prompt → 单次API调用 → 输出
V3: 用户输入 → Few-shot增强Prompt → 单次API调用 → 输出
V4: 用户输入 → 案例库检索 → 动态Prompt → 生成初稿 → 规则评分 → AI审稿 → 精修输出
                    ↑                                                         ↓
              用户反馈/偏好 ←←←←←←←←←←← SQLite持久化 ←←←←←←←←←←←←← 保存结果
```

### Agent 三轮迭代机制

| 轮次 | 角色 | 输入 | 输出 |
|------|------|------|------|
| Round 1 | 创作者 | 用户需求 + 检索案例 + 用户偏好 | 初稿 |
| Round 2 | 审核官 | 初稿 + 5维度规则评分 | 审稿意见（问题+改进指令） |
| Round 3 | 创作者 | 初稿 + 审稿意见 | 精修终稿（流式输出给用户） |
