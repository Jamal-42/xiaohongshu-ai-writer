# 需求分析文档

## 一、项目背景

小红书作为国内领先的生活方式社区，日活用户超过1亿。大量个人博主和品牌运营需要持续产出高质量内容，但面临以下痛点：

- 日更压力大，创作灵感枯竭
- 平台文案风格独特（emoji密集、口语化、结构化），新手难以掌握
- 标题和标签的选择直接影响流量，试错成本高
- 缺乏数据参考，不知道什么样的内容容易火

## 二、目标用户

| 用户画像 | 描述 | 核心诉求 |
|---------|------|---------|
| 个人博主 | 需要日更，时间有限 | 快速产出高质量初稿 |
| 小型品牌运营 | 1-2人团队，多品类覆盖 | 批量生产不同风格文案 |
| 新手创作者 | 不熟悉平台调性 | 学习爆款写作模式 |

## 三、核心功能

### 3.1 智能文案生成
- 输入：主题关键词 + 补充信息
- 输出：3个备选标题 + 正文 + 互动引导 + 推荐标签
- AI 自动检索相关爆款案例作为参考，多轮迭代后输出最终版本

### 3.2 创作参数配置
| 配置 | 选项 |
|------|------|
| 文案风格 | 种草推荐 / 产品测评 / 教程攻略 / 日常分享 / 避雷吐槽 / 好物合集 |
| 语气调性 | 活泼俏皮 / 专业理性 / 温柔治愈 / 犀利直白 / 幽默搞笑 |
| 目标受众 | 18-25岁学生党 / 25-30岁职场新人 / 30-35岁精致妈妈 / 泛人群 |
| Emoji密度 | 5级可调（纯文字 → 每句都有） |
| 文案长度 | 短(100-200字) / 中(200-400字) / 长(400-600字) |

### 3.3 智能优化机制
- 爆款案例库检索：根据输入主题匹配最相关的高互动案例
- 多轮 Agent 迭代：生成 → 质量评估 → 审稿 → 精修
- 规则评分引擎：5维度量化评分（标题吸引力/信息密度/情绪价值/互动潜力/平台适配度）

### 3.4 个性化学习
- 历史记录持久化存储
- 用户反馈机制（满意度 + 改进建议）
- 偏好学习，影响后续生成质量

## 四、非功能需求

- 响应体验：流式输出，生成过程中展示 AI 思考步骤
- 数据安全：API Key 环境变量管理，不暴露在前端
- 部署方式：本地运行，代码托管至 GitHub
- 技术约束：Python + Streamlit + 通义千问 API + SQLite
