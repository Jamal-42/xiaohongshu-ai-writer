# 需求分析文档

## 一、项目背景

小红书作为国内领先的生活方式社区，日活用户超过1亿。大量个人博主和品牌运营需要持续产出高质量内容，但面临以下痛点：

- 日更压力大，创作灵感枯竭
- 平台文案风格独特（emoji密集、口语化、结构化），新手难以掌握
- 标题和标签的选择直接影响流量，试错成本高

## 二、目标用户

| 用户画像 | 描述 |
|---------|------|
| 个人博主 | 需要日更，时间有限，希望快速产出初稿 |
| 小型品牌运营 | 1-2人团队，需要批量生产种草文案 |
| 新手创作者 | 不熟悉小红书文案风格，需要参考模板 |

## 三、核心功能

### 3.1 文案生成（MVP）
- 输入：主题关键词 + 补充信息
- 输出：3个备选标题 + 正文 + 推荐标签
- 可配置：风格、语气、emoji密度、字数

### 3.2 配置项
| 配置 | 选项 |
|------|------|
| 文案风格 | 种草推荐 / 产品测评 / 教程攻略 / 日常分享 |
| 语气 | 活泼俏皮 / 专业理性 / 温柔治愈 |
| Emoji密度 | 1-5级可调 |
| 文案长度 | 短/中/长 |

## 四、非功能需求

- 响应时间：单次生成 < 15秒
- 部署方式：本地运行，支持推送至 GitHub
- 技术约束：Python + Streamlit + 通义千问 API
